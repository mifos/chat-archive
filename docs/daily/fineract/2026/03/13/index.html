If you look around at other Apache projects, they typically use the SIPs (Significant Improvement Projects) for decisions that need a deliberation because of the impact on the architecture or trajectory of the project.

  • Releases already have a VOTE.
  • Tickets and code changes we have code reviews.
  • Less significant changes we have a norm of "lazy consensus".... if no one objects in 72 hrs, you can do the thing you proposed.
